2186  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS ONLY

2193  * a ready list. If a non-zero value is returned then a context switch is
2194  * required because either:
2195  * + A task was removed from a blocked list because its timeout had expired,
2196  * or

2198  * currently running task.
2199  */

2202 /*
2203  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
2204  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
2205  *
2206  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
2207  *
2208  * Removes the calling task from the ready list and places it both
2209  * on the list of tasks waiting for a particular event, and the
2210  * list of delayed tasks. The task will be removed from both lists
2211  * and replaced on the ready list should either the event occur (and
2212  * there be no higher priority tasks waiting on the same event) or
2213  * the delay period expires.
2214  *
2215  * The 'unordered' version replaces the event list item value with the
2216  * xItemValue value, and inserts the list item at the end of the list.
2217  *
2218  * The 'ordered' version uses the existing event list item value (which is the
2219  * owning tasks priority) to insert the list item into the event list is task
2220  * priority order.
2221  *
2222  * @param pxEventList The list containing tasks that are blocked waiting
2223  * for the event to occur.
2224  *
2225  * @param xItemValue The item value to use for the event list item when the
2226  * event list is not ordered by task priority.
2227  *
2228  * @param xTicksToWait The maximum amount of time that the task should wait
2229  * for the event to occur. This is specified in kernel ticks,the constant
2230  * portTICK_PERIOD_MS can be used to convert kernel ticks into a real time
2231  * period.
2232  */
2233 void vTaskPlaceOnEventList( List_t * const pxEventList, const TickType_t xTicksToWait ) PRIVILEGED_FUNCTION;
